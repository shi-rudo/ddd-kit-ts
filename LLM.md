# @shirudo/ddd-kit

> Composable TypeScript toolkit for tactical Domain-Driven Design. Ships the canonical building blocks (Value Objects, Entities, Aggregate Roots, Domain Events, Repositories, and CQRS handlers) without a framework or runtime lock-in. Targets Node 22+, Cloudflare Workers, Vercel Edge, Deno, and Bun. ESM-only.

The library is **2.0, stable**. The public surface is under Semantic Versioning; breaking changes bump the major and ship with a migration path. Per-release changes and migrations are documented in [CHANGELOG.md](https://github.com/shi-rudo/ddd-kit-ts/blob/main/CHANGELOG.md); read the latest entry before upgrading across a major.

Architectural choices to know before reading the rest:

- **Domain layer throws `DomainError`-derived exceptions; App-Service boundary chooses `Result`.** Aggregates / entity validators / value-object constructors throw on invariant violations (Vernon-canonical). `CommandBus.execute` and `QueryBus.execute` carry explicit handler failures in `Result<T, E>`; handler throws propagate unchanged unless optional `mapExpectedError` positively recognizes one by returning `{ error }` (`undefined` means rethrow). `withCommit` returns its `result` value directly, while `loadFromHistory` uses `Result` because event-stream corruption is a recoverable infrastructure failure.
- **Two error styles, distinguished by how you consume them.** `DomainError` is thrown and caught at the boundary; `ValidationError` (from `@shirudo/base-error`) is a Result-axis *value* you destructure, never catch. `voValidated` collects multiple field violations into one `ValidationError`; the opt-in `@shirudo/ddd-kit/http` entry renders it as RFC 9457. See Result vs Throw.
- **Result type comes from `@shirudo/result`** as a peer dependency. Not re-exported.
- **Error hierarchy** uses `@shirudo/base-error` as a peer dependency. `DomainError<Name>` and `InfrastructureError<Name>` extend `BaseError<Name>` directly. Library-internal concrete classes: `AggregateNotFoundError`, `ConcurrencyConflictError` (retryable, in a FRESH unit of work), `DuplicateAggregateError` (insert hit an existing id; map your driver's unique-violation: Postgres 23505, MySQL 1062, SQLite SQLITE_CONSTRAINT_UNIQUE; NOT retryable) and the UoW lifecycle wrappers `CommitError`/`RollbackError` as infrastructure errors (all five are `InfrastructureError` subclasses), plus the crash-loud programming-bug classes `MissingHandlerError`, `AggregateDeletedError`, `NestedUnitOfWorkError`, `TransactionClosedError`, `EventHarvestError`, `UnenrolledChangesError` (BaseError-direct, so generic infrastructure handlers cannot mask them; `EventHarvestError` is the deterministic harvest-guard failure, kept off `InfrastructureError` so retry paths skip it; `UnenrolledChangesError` is the end-of-run guard for a loaded aggregate that recorded events but was never enrolled). All pin their runtime `name` (minification- and subclass-stable).
- **Aggregates are class-based.** Extend `AggregateRoot` (state stored) or `EventSourcedAggregate` (state derived from events). There is no functional aggregate API. Both have `protected constructor`; construction goes through static factory methods (`Order.place(...)`, Vernon §11 Aggregate Factory). Reconstitution from persistence is a separate static method (`Order.reconstitute(id, state, version)`); see the Reconstitution section in the aggregates guide.
- **`TEvent` defaults to `never`** on both aggregate flavours. Aggregates that emit events MUST declare the event union explicitly: `class Order extends AggregateRoot<OrderState, OrderId, OrderEvent>`. Forgetting the third generic makes every `addDomainEvent` / `commit(state, event)` call a type error ("argument not assignable to `never`"). This is the most common compile-time footgun newcomers hit.
- **`withCommit` owns the post-save lifecycle.** `Repository.save` is **pure persistence**: it writes the aggregate row(s) and throws `ConcurrencyConflictError` on OCC mismatch, nothing else. The callback receives an invocation-scoped `CommitEnrollment` and returns `{ result, commits: [...] }` with every opaque token issued by `enrollSaved` / `enrollDeleted`; naked aggregates, forged tokens, tokens from earlier invocations, and omitted issued tokens are rejected before harvest. `withCommit` harvests `pendingEvents` from each enrolled aggregate inside the transaction, writes to `Outbox.add` atomically, commits, then calls `markPersisted` for saved enrollments (deleted enrollments are cleared without the post-save hook), then publishes to `EventBus` if wired.
- **`markPersisted` is framework-internal.** Subclasses should override the `onPersisted(version)` Template-Method hook instead; overriding `markPersisted` directly without `super` silently leaks `pendingEvents` and double-dispatches events on the next save.
- **`Repository` must maintain an Identity Map** (Fowler PoEAA): two `getById(id)` calls within the same Unit of Work return the **same in-memory instance**. Commit enrollment dedupes by JS object identity; distinct instances with the same logical id can still receive distinct tokens. The `UnitOfWork` facade ships the map as `session.identityMap`; hand-rolling applies to `withCommit`-only setups.
- **Event sourcing structurally enforces "record after mutation".** `EventSourcedAggregate.apply()` is atomic: state, event, version commit together; a throwing handler mutates nothing. Aggregate version IS event count (Greg Young / Vernon §9 canonical).
- **Repository contract stops at identity and lifecycle.** `IRepository<TAgg, TId>` is id-canonical CRUD. Consumers declare intent-revealing command-side query ports; single-result methods state their uniqueness law, while multi-result methods require validated bounds, stable ordering, and cursor semantics. SQL, Prisma `WhereInput`, Mongo filters, and other adapter-native query languages do not cross that port.
- **In-memory `CommandBus` / `QueryBus`** are zero-config in-process dispatchers: designed for edge runtimes (one Worker invocation), modular monoliths, tests. For cross-process messaging use the `CommandHandler<C,R>` / `QueryHandler<Q,R>` types with your own broker. `InMemoryOutbox<Evt>` ships as the canonical Map-backed reference outbox for tests + single-process workers.
- **Event-port constraints are tight.** `EventBus<Evt>`, `Outbox<Evt>`, `OutboxRecord<Evt>`, `withCommit<Evt>` all require `Evt extends AnyDomainEvent` (alias for `DomainEvent<string, unknown>`). The library is for *domain* events; non-event tagged unions are rejected at the type level.
- **Process Managers are aggregates.** A saga / process manager extends `AggregateRoot` (Vernon §12-13); its outputs are commands dispatched via `CommandBus`, not events published via `EventBus`. See `examples/saga/` for the worked example.

## Installation

```bash
pnpm add @shirudo/ddd-kit @shirudo/result @shirudo/base-error
```

Both `@shirudo/result` and `@shirudo/base-error` are peer dependencies; install them once in the consuming app.

## API surface (high signal)

- `src/core/errors.ts`: `DomainError`, `InfrastructureError`, `MissingHandlerError`, `AggregateDeletedError`, `AggregateNotFoundError`, `ConcurrencyConflictError` (`retryable: true`), `DuplicateAggregateError` (insert-path unique-violation mapping, not retryable), plus `KitErrorOptions` and the `KitErrorCode` union. All are `StructuredError`s from `@shirudo/base-error`: one stable SCREAMING_SNAKE `code` per error, `error.name === error.code`, `category` derived from the hierarchy (DOMAIN / INFRASTRUCTURE / WIRING). Consumers need no base-error import: plain `switch (error.code)` plus the kit-exported `instanceof` bases cover routing; `matchError`/catalogs are opt-in on top. Consumer subclasses pass `{ code, message, cause?, retryable? }` to `super`.
- `src/core/id.ts`: `Id<Tag>` brand and `IdGenerator<Tag>`. JSDoc spells out collision-resistance requirements (safe: UUIDv4/ULID/UUIDv7/KSUID; unsafe: `Date.now()` alone, non-persistent counters).
- `src/aggregate/aggregate.ts`: `Version` brand, `AggregateSnapshot`, `sameVersion`, plus re-exports from domain-event.
- `src/aggregate/aggregate-root.ts`: `AggregateRoot`, `IAggregateRoot<TId, TEvent>`, `AggregateConfig`. `commit(state, events)` always bumps version. `pendingEvents` + `clearPendingEvents` on the interface. `protected onPersisted(version)` Template-Method hook (the safe subclass extension point). Dirty tracking for partial writes: `changedKeys: ReadonlySet<keyof TState & string>` (shallow per-key reference diff against the state captured at `markRestored` / `markPersisted`; never-persisted → all keys) and `hasChanges` (true when never persisted, `version !== persistedVersion`, unflushed `pendingEvents`, any key dirty, or (for keyless states the diff cannot see) the state reference changed: the safe skip-save signal). Lives on `AggregateRoot` only, NOT on `IAggregateRoot` or `EventSourcedAggregate` (whose `pendingEvents` are its change record); the per-key diff is exact only for plain-record `TState` mutated via `setState` / `commit`. See repository guide → "Partial writes for multi-table aggregates".
- `src/aggregate/event-sourced-aggregate.ts`: `EventSourcedAggregate`, atomic `apply()`, `loadFromHistory(history: ReadonlyArray<TEvent>)` returns `Result<void, DomainError>` and IS the reconstitution path. `protected onPersisted(version)` hook (mirrors AggregateRoot).
- Snapshots: snapshot state MUST be plain data; the default `createSnapshot` fails fast (with path) on class instances / functions / uncloneables in state. For class-based child entities, declare a plain DTO via the trailing `TSnapshotState` generic and override `protected toSnapshotState(state)` / `fromSnapshotState(stored)` (on `BaseAggregate`, honoured by `restoreFromSnapshot` and `restoreFromSnapshotWithEvents`).
- `src/aggregate/domain-event.ts`: `DomainEvent<T, P>` (fields incl. `version` = SCHEMA version for upcasting and the distinct optional `aggregateVersion` = producing aggregate's commit version and `commitSequence` = zero-based index within the aggregate's harvest batch, both stamped by withCommit at harvest; the pair is a total order per aggregate and a compact consumer watermark), `AnyDomainEvent` alias (= `DomainEvent<string, unknown>`), `createDomainEvent`, `EventIdFactory`, `ClockFactory`, `setEventIdFactory` / `setClockFactory` globals, **`withEventIdFactory` / `withClockFactory` scoped helpers** with thenable-guard, metadata helpers (`copyMetadata`, `mergeMetadata`). Events are deeply frozen at construction.
- `src/entity/entity.ts`: `Entity`, `Identifiable`, `IEntity`, `freezeShallow`, entity-collection helpers (`findEntityById`, `hasEntityId`, `removeEntityById`, `updateEntityById`, `replaceEntityById`, `entityIds`); all accept `ReadonlyArray<T>`.
- `src/value-object/value-object.ts`: `vo`, `voEquals`, `voEqualsExcept`, `voWithValidation`, `ValueObject` base, `deepFreeze`.
- `src/validation/`: `voValidated(t, (issues, value) => …, message?)` builds a frozen VO while collecting **all** field violations into one base-error `ValidationError`; returns `Result<VO<T>, ValidationError>` (the failure is a value, not a throw). `ValidationError` is imported from `@shirudo/base-error`, not re-exported.
- `src/http.ts` (subpath `@shirudo/ddd-kit/http`): `toProblemDetails(error: ValidationError, { status?, type?, title?, detail?, instance?, extensions? })` delegates to base-error's `toProblem` transport stage and returns its `ProblemDetailsResult` (`{ status, headers, body, outcome }`): the body carries the error's public `code` and `publicIssues()` under `details.issues` (same shape as `toPublicErrorView`), deeply frozen, null-prototype, JSON-safe, defaulting to 422/"Validation Failed". One pipeline, one wire profile, one hardening implementation; for catalog-driven mapping use `definePublicErrors` + `project` + `toProblem` directly. Opt-in so transport stays out of the core barrel.
- `src/presentation.ts` (subpath `@shirudo/ddd-kit/presentation`): `toPublicErrorView(error, { locale?, catalog? })` maps a kit error (or any caught value) to a base-error `LocalizedPublicError` (`{ code, message, locale, details? }`) by delegating to the public-error pipeline (`project` + `localize`); `locale` is a preference and the view carries the RESOLVED locale. `createKitPublicErrors()` builds the kit catalog (a factory, not a shared instance: registrations mutate the instance) with per-code transports and messages; consumers extend it via `registerByCode`/`register` and pass it as `catalog` (and to `toProblem` for per-code status mapping). NO status/header/exit-code (the consumer's transport adapter owns those); feed the view into base-error's `toProblem` (HTTP), a gRPC table, or a CLI exit-code map. Total over `unknown` (unmapped → generic `INTERNAL_ERROR`); kit errors matched by their stable `code` (which is also `error.name`), `ValidationError` by its `publicIssues()` whitelist (issues in `details.issues`). Opt-in home for the client-safe messages 2.0 removed from the core; the core stays presentation-free.
- `src/money.ts` (subpath `@shirudo/ddd-kit/money`): the canonical money CONTRACT and its boundaries; exact operations only (all lossy math, allocation, and FX stay with calculation libraries as domain-named policy; the kit bridges to them). `Money` = `{ amountMinor: bigint, currency: string, scale: number }`, frozen plain data, safe for aggregate state / event payloads / snapshots (survives structuredClone and the kit's deep-freeze and diffing); `MoneyDto` carries `amountMinor` as an integer STRING for JSON safety (`moneyFromDto` validates `/^-?\d+$/`, `moneyToDto` emits and guards). Exact operations ONLY: lossless `addMoney`/`subtractMoney` (exact on aligned minor units, same currency AND scale enforced via `MONEY_CURRENCY_MISMATCH`/`MONEY_SCALE_MISMATCH`), `negateMoney`, and lossless `rescaleMoney` (up lossless, exact down, inexact -> `MONEY_PRECISION_LOSS`; no rounding parameter). The kit NEVER rounds, anywhere. NOT shipped, deliberately: anything with a rounding or distribution POLICY (multiplication, ratios, fees, division, lossy rescaling, allocation, FX); the domain names the policy (calculateVat(net, vatPolicy)), the consumer's calculation library executes it via the snapshot bridge. Domain code needs no third-party import. `Money` is BRANDED with a NON-EXPORTED unique symbol: structural literals do not type-check (a spelled-out __brand property does not help), only the constructors mint it. `moneyOfMinor(amountMinor, currency, scale)` is the typed constructor (rejects number amounts, bad scales, whitespace currencies with `INVALID_MONEY`); `moneyFromUnknown` re-hydrates foreign plain shapes (validates, COPIES, freezes: input mutation cannot reach domain state); the trust-boundary parsers `moneyFromDto`/`parseMoneyInput`/`moneyFromSnapshot`/`factory.parse` take `unknown` (no caller casts, no String() coercion); `isMoney` narrows but is a check, not a door (no copy, no freeze); `moneyEquals` is strict on amount AND currency AND scale; predicates `isZeroMoney`/`isPositiveMoney`/`isNegativeMoney`. `parseMoneyInput(input, { currency, scale })`: EXACT OR REJECTED decimal-string parsing (grammar `/^-?\d+(\.\d+)?$/`, no floats ever, the safe replacement for `Number(x)*100`); missing fraction digits pad losslessly, all-zero excess is lossless, real excess throws `MONEY_PRECISION_LOSS`. The kit ships NO rounding at all (no modes, no engine); whether over-precise input is rejected or rounded is a business decision that belongs in a domain-named policy function using the calculation library; `moneyToDecimalString` is the exact inverse. `createMoneyFactory({ scaleFor })` binds a `CurrencyScaleResolver` once (kit ships NO currency table) and returns `ofMinor`/`parse`/`zero`/`scaleOf` (unresolved -> `UNKNOWN_CURRENCY`); resolvers: `currencyScaleFromRecord({ EUR: 2 })`, `currencyScaleFromIntl()` (runtime ICU data; canonical uppercase ISO codes ONLY, 'eur' -> undefined instead of silently aliasing 'EUR'; a CONVENIENCE, not an enterprise source of truth: well-formed unassigned codes resolve to 2 and the data shifts with the ICU version, production pins a closed versioned map), or a one-liner over a calculation library's currency package. `moneyFromSnapshot`/`moneyToSnapshot` bridge the structural `{ amount, currency, scale }` snapshot shape of calculation libraries with anti-corruption checks (non-base-10 currencies rejected, number amounts must be safe integers, bigint passes exactly; `moneyToSnapshot` is number-based and refuses amounts past 2^53). `formatMoney(money, locale)` / `createMoneyFormatter(locale)` (cached, cache bounded) feed Intl the exact decimal STRING (precision-safe past 2^53), presentation only; both guard non-Money input loudly, as do `moneyToDto`, `moneyToSnapshot`, and `moneyToDecimalString`. Hard bounds by construction: amounts below 97 digits (uint256 fits), scale <= 64, currency <= 32 chars, parse input <= 256 chars; oversized/hostile input is INVALID_MONEY in O(1) (length prechecks run before BigInt conversion) and diagnostic messages truncate what they echo; resolver/formatter caches are size-capped against attacker-influenced keys. Errors are kit StructuredErrors; `INVALID_MONEY`, `MONEY_PRECISION_LOSS`, `UNKNOWN_CURRENCY` are DomainErrors registered in `createKitPublicErrors()` at 422. FX ships as a documented practice (audit-record shape in the guide), not as code.
- `src/testing.ts` (subpath `@shirudo/ddd-kit/testing`): **`createRepositoryContractTests(harness): RepositoryContractTest[]`**: the repository contract test suite. OCC is a repository contract, not a kit guarantee; this suite is the proof an adapter must pass. Framework-agnostic (plain-Error assertions; bind to vitest/jest/node:test via `it(test.name, test.run)`). Error matching is by NAME along the cause chain, never `instanceof`, robust across bundle entries and duplicate kit installs. Harness carries the aggregate factories/mutators (`createAggregate`, `mutate` = exactly one version-bumping event-recording mutation) and optional capabilities (`mutateVersionOnly`, `mutateChildCollection`, `createAggregateWithId` for the deletion-finality-across-instances AND duplicate-insert tests (`DuplicateAggregateError` on an existing id, existing row untouched; `insertsAreDuplicateChecked: false` is the explicit opt-out for deliberately upserting adapters; save() is insert-or-update, never upsert; create-idempotency belongs in the use case), `snapshotState` for the mandatory and duplicate-insert tests' deep-equal state assertions, must be roundtrip-stable, `deletesAreVersionChecked` for the stale-delete conflict test). Tests for absent capabilities come back MARKED `skipped` with a loudly-failing run() (bind `(test.skipped ? it.skip : it)(test.name, test.run)`); capabilities are captured at suite creation; `createEnvironment()` yields per-test isolated environments (`run` through the adapter's real UnitOfWork, `committedOutboxEvents`, `teardown`; teardown failures never mask the contract diagnostic). Mandatory two-writer test: stale writer must reject with `ConcurrencyConflictError` (by its stable name/code `CONCURRENCY_CONFLICT` anywhere in the chain; kit errors are StructuredErrors whose `name === code`, minification- and subclass-stable; failure messages render the cause-chain names, so unexpected errors (raw driver errors, a different kit copy in the dependency graph) are identifiable from the diagnostic), final version+state = winner's, outbox = exactly the winner's events (eventIds compared as a sorted multiset, no ordering requirement on committedOutboxEvents). env.run MUST provide unit-of-work semantics (identity map + deletion gates). Known limitation: sequential-deterministic; lock interaction (FOR-UPDATE blocking, raw 40001 mapping) needs adapter-specific tests. SQL/ORM adapters MUST run against a real database; the in-memory reference adapter in `src/testing/repository-contract.test.ts` (repo-only, not shipped) is the copyable example: identity-map read path with isDeleted, enroll-before-write, real predicates on update AND delete, snapshot/restore transactionality.
- `src/events/ports.ts`: `EventBus<Evt extends AnyDomainEvent>`, `OnceOptions`, `Outbox<Evt extends AnyDomainEvent>`, `OutboxRecord<Evt extends AnyDomainEvent>`, `EventHandler<Evt>`.
- `src/events/event-bus.ts`: `EventBusImpl<Evt extends AnyDomainEvent>` (publish ordering: input order across events, parallel within event via `Promise.allSettled`).
- `src/events/outbox.ts`: `InMemoryOutbox<Evt extends AnyDomainEvent>` Map-backed reference implementation, idempotent on duplicate add by eventId.
- `src/app/command.ts`, `command-bus.ts`, `query.ts`, `query-bus.ts`, `handler.ts`: `Command`, `Query`, `CommandHandler`, `QueryHandler`, `CommandBus<TMap>`, `QueryBus<TMap>`. Bus error policy is selective: returned command `err(E)` values pass through; optional `mapExpectedError(thrown)` must return `{ error: E }` only for a recognized expected failure, while absence or `undefined` rethrows the exact original. Unknown programmer, cancellation, infrastructure, unregistered-handler, and nested mapper/wiring failures therefore stay throws by default. **`withCommit<Evt extends AnyDomainEvent, R, TCtx>`** takes `fn: (ctx, enrollment) => Promise<{ result: R; commits: ReadonlyArray<AggregateCommitToken<Evt>> }>`. `CommitEnrollment.enrollSaved/enrollDeleted` mints opaque, invocation-bound tokens; a missing, forged, or stale token fails inside the transaction, so an unsaved aggregate cannot be harvested or marked persisted through a naked return value. Tokens are idempotent by aggregate reference and define harvest order. `withCommit` harvests pending events inside the tx, composes frozen commit candidates with source and position, calls `markPersisted` post-commit for saved enrollments (deleted enrollments clear pending events without the post-save hook), then publishes to the bus. A post-commit `bus.publish` failure NEVER rejects `withCommit`; the committed `result` is always returned and the error goes to the optional `onPublishError(error, events)` dep.
- `src/app/unit-of-work.ts`: **`UnitOfWork<Evt, TCtx, TRepos>`** with `run(work)`: explicit-save UoW facade over `withCommit`. Deps: `{ scope, outbox, bus?, onPublishError?, repositories: RepositoryFactories }` (factory map `(tx, session) => repo`, all factories called once per run with the SAME tx). Callback context: `{ repositories, rawTransaction, session }`. All throw `TransactionClosedError` after close; `rawTransaction` is the deliberately-ugly escape hatch (bypasses enrollment AND identity map; pair with manual `session.enrollSaved`). OCC is a REPOSITORY CONTRACT, not a kit guarantee: the kit ships boundary/baseline/error/pattern, the consumer repo must implement `WHERE version = persistedVersion`; `version` is a mutation sequence (3 mutations = +3), not a commit revision; the predicate always uses the load-time `persistedVersion`. No external side effects inside the transaction (outbox-first); aggregates must not be reused after a rollback. Repos enroll written aggregates via `UnitOfWorkSession.enrollSaved/enrollDeleted` (idempotent per instance; deleted aggregates still harvested; save-after-delete throws `AggregateDeletedError`). Callback errors pass through UNCHANGED (`ConcurrencyConflictError` stays distinguishable); `CommitError` wraps the potentially-transient post-callback failures (outbox write, commit); a deterministic harvest-guard violation surfaces as `EventHarvestError` (BaseError-direct, not retryable) passed through `run()` unchanged (cause-chain-unwrapped even when a wrapping scope nests it), `RollbackError` wraps callback-failed-AND-scope-rejected-differently (primary error as cause, scope error in `rollbackCause`; wrapper-scopes detected via cause chain and passed through). One instance = one operation at a time (`NestedUnitOfWorkError` on nesting/concurrent sharing; sequential reuse fine). `session.identityMap` is a fresh per-run `IdentityMap`, cleared on close. **`IUnitOfWorkRepository`** (defined in src/repo/repository.ts) is the canonical UoW-facing repo shape; the contract suite's `ContractRepository` is its minimal structural subset. HARD RULE: a repository write rejection aborts the unit of work; never catch and continue in the same run() (the failed aggregate is already enrolled; retry = fresh run()). No auto-flush, no savepoints.
- `src/repo/identity-map.ts`: **`IdentityMap`**: per-unit-of-work Fowler Identity Map. Two-level storage keyed on the aggregate CLASS (`AggregateClass<TAgg>` accepts protected-constructor classes via a prototype-witness branch, the kit's aggregate convention), so `Restaurant:123` / `Booking:123` never collide and `get` returns `TAgg` without casts. `set` is strict: same-instance re-set is a no-op, different instance for an occupied type+id throws (the hydrated-twice violation), and a type+id deleted in this unit of work throws `AggregateDeletedError` (deletion tombstones survive until `clear()`). `isDeleted(Type, id)` distinguishes deleted-in-this-UoW from never-loaded so the read path returns null uniformly. Repos: check `get` before hydrating, treat `isDeleted` as not-found, `set` after; on delete `session.enrollDeleted(aggregate)` is the ONE call: it removes+tombstones the map entry automatically (keyed on the instance's concrete class).
- `src/repo/repository.ts`: `IRepository<TAgg, TId>` and **`IUnitOfWorkRepository<TAgg, TId, Evt>`** (canonical UoW-facing shape: findById/getById/save(agg)/delete(agg), branded ids). No generic filtered repository is exported; consumer applications own domain-specific query ports. `save(agg): Promise<void>` is pure persistence (must NOT call `markPersisted`). JSDoc covers the insert-vs-update convention (`persistedVersion === undefined` → INSERT; otherwise UPDATE with `WHERE version = persistedVersion` as the OCC baseline; do NOT route on `version === 0`) and deletion patterns.
- `src/repo/scope.ts`: `TransactionScope<TCtx>` generic, no default for `TCtx` (use `TransactionScope<undefined>` for context-free / in-memory scopes); `transactional(fn, options?: { signal? })` (additive optional opts, one-arg impls still satisfy it).
- `src/repo/retrying-scope.ts`: **`RetryingTransactionScope<TCtx>`** wraps an inner scope + `RetryPolicy` `{ maxAttempts=3, baseDelayMs=50, maxDelayMs=1000, isRetryable=someChainRetryable, onRetry?, sleep?, random? }`. Retries only classifier-accepted errors (default `someChainRetryable`, so `ConcurrencyConflictError` retries even when wrapped; deterministic kit errors surface immediately), exponential backoff with +/-20% jitter band capped at `maxDelayMs`, rethrows last error UNCHANGED after `maxAttempts`, bounded by the `transactional` AbortSignal (no separate max-elapsed knob). Retries the transaction only; the work callback must be reload-safe and side-effect-free before commit. Compose as the UoW's scope. Pure `computeBackoffDelay(attempt, {baseDelayMs,maxDelayMs,random})` exported for testing.

## Optional

- [@shirudo/result](https://www.npmjs.com/package/@shirudo/result): Result/Either implementation, peer dependency. `_tag: 'Ok' | 'Err'`, method-style guards (`result.isOk()`), curried operators (`flatMap`, `mapAsync`, `pipeAsync`, etc.).
- [@shirudo/base-error](https://www.npmjs.com/package/@shirudo/base-error): Base error class, peer dependency (`^8.0.0`, requires Node `>=22`). Provides `BaseError<Name>`, `toJSON` / `toLogObject` (log shape, never client-facing), `isBaseError`, `isRetryable`, `someChainRetryable`, `someCauseChain`, `getRootCause`, `findInCauseChain`, `filterCauseChain`, plus the `ValidationError` aggregate (field-level issues, Standard Schema ingestion, `publicIssues()` whitelist). v6 made the core purely technical: the user-message/localization API (`withUserMessage` / `getUserMessage` / `addLocalizedMessage`) and the in-core serializers (`toPublicJSON` / `toProblemDetails` / `toErrorResponse`) were removed. Client-safe, localized output now lives in the opt-in `@shirudo/base-error/public-error` subpath (`the public-error projection pipeline`); RFC 9457 mapping lives in `@shirudo/base-error/public-error` (`ProblemDetails` type + `toProblem`). The kit builds `voValidated` and the `/http` presenter on these. Strict-shape helpers (`isChainRetryable`, `isStructuredError`, `isRetryableStructuredError`, `getRootCauseRetryable`, `getFirstRetryableCause`) target `StructuredError`-shaped errors and since v3 match every ddd-kit error (kit errors ARE StructuredErrors); `someChainRetryable` remains the tolerant default classifier.
